C4K会議_第10回

2020-11-11 Code for Kanoya

1 分で読めます

先日、定例ミーティングを開催しました😃協議事項は以下のとおり🙂
①アプリの開発は進んだのですが、動作する速度が出ていません😕コーディングを見直す必要がありそうです🧐
②オープンデータのクローリング方法も試行錯誤しているのですが、取得できるデータセットの数があまり増えません😢原因を探るために手動でBODIKを覗いてみると、各データセットに対する命名方法に問題がありそうです🤨
③例えば鉄道駅の一覧を取得したい場合🤠「鉄道駅」と親切に命名してくれているものはないので、「駅」で検索するしかありません😔そうすると、道の駅やら赤ちゃんの駅やら不要なデータセットが多数引っ掛かってしまいます😲解決策を検討していきたいと思います🤯

C4K会議_第9回

2020-10-27 Code for Kanoya

1 分で読めます

昨日、定例ミーティングを開催しました😃今回も福岡から1名のメンバーが参加してくれました😮協議事項は以下のとおり🙂
①アプリの開発は着々と進んでいます🧐OpenStreetMap上でデータを可視化できるようになっていたので、とても汎用性が高そうです😊
②オープンデータのクローリング方法も一部変更しました🤠その結果、九州の自治体数234に対して88自治体のオープンデータが取得できるようになりました😉とは言っても、やはり九州はオープンデータ後進地域ですね😤
③取得できたデータ数は、本来1,320あるべきところ、188でした😕44件はエラー、1,088件はそもそもアップロードされていないという状況😢しかし取得できたデータでできるところまでやるしかありませんので、頑張ります😘

C4K会議_第8回

2020-10-15 Code for Kanoya

1 分で読めます

先日、定例ミーティングを開催しました😃今回は肝付から新たに1名、福岡から1名のメンバーが参加してくれました😮協議事項は以下のとおり🙂
①今回はアプリの開発について進捗はありませんでした🧐メンバーの本業の繁閑状況に配慮しながらやっていければ良いと思っています😊
②そこでオープンデータの内容について詳しく見ていくことにしました😕まずはファイル名が様々であることについて、データセット名と同一でなくてもダウンロードして問題ないことを確認しました😉
③次に、1つのデータセットに対して複数のファイルがアップロードされていることについて、ルール違反であることを確認しました🤠どれが正しいファイルであるかは機械判別できないため、最上段のファイルをダウンロードして内容を確認することとしました😤
今回新たに参加してくれたメンバーからの意見が非常に助けになり、感謝です😘
